There was never a product spec for belal.work. Each tool started the same way: I needed to convert something, sign something, count something, and every option I found online wanted an account, an upload, or fifteen seconds of ads before it would let me do a thirty-second task. So I built the version I wanted, put it on a subdomain, and moved on.

That's now eleven tools — a PDF signer, an image converter, a word counter, a Markdown-to-Word exporter, a color palette generator, and more — with nothing tying them together except the same domain and the same rule: free, fast, and it runs in your browser.

## No roadmap, on purpose

I don't keep a backlog for belal.work. A tool gets built when I hit a real annoyance that a thirty-minute build can fix. That's why the suite is uneven — some tools are polished products with their own docs and changelog, others are a single focused screen that does exactly one thing. Both are fine. The bar isn't "is this a complete product," it's "does this remove a real annoyance, for free, without asking for my email address."

If you're looking for what's next: I don't know yet either. That's kind of the point.
